# Justificación del Modelado UML

Los diagramas de Casos de Uso representan las interacciones principales entre los actores y el sistema, alineadas con los requisitos funcionales definidos.

El diagrama de Clases modela las entidades principales del sistema, sus atributos y relaciones, asegurando coherencia con los procesos de negocio y facilitando la existencia de un implementación futura.

Ambos diagramas permiten validar la arquitectura lógica del sistema con las historias de usuario.
